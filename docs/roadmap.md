# ロードマップ（次期イテレーション計画）

最終更新: 2026-07-10

本ドキュメントは現状（ベジ探アダプタによる野菜卸売価格の自動更新サイト、AdSense/アフィリエイトは
未設定）を起点に、次に着手すべき作業をフェーズ分けして記す。各フェーズに完了条件と
「自動化度」（GitHub Actions で完結する部分と、人力（オーナー）作業が必要な部分の区別）を明記する。

## フェーズA: 法務・基盤（一次ソースへの移行）

**目的**: `docs/legal-notes.md` で整理したリスク（データベース著作物性・利用規約上の未対応事項）を
実質的に解消するため、政府標準利用規約準拠の一次統計に移行する。

### A-1. e-Stat「青果物卸売市場調査」（農林水産省・月次）のデータURL特定
- 内容: e-Stat の統計表ID・CSV直リンクまたは API（appId方式）のエンドポイントを確定し、実データの
  列レイアウトを確認する。
- 手段: `scripts/probe.mjs` の候補URLに e-Stat 系ホストを追加し、本番ランナー（GitHub Actions）で
  probe ワークフローを実行、実レスポンスをフィクスチャとして保存する
  （ビルドサンドボックスは egress 403 のため直接確認不可。`docs/data-sources.md` 参照）。
- 完了条件: 実レスポンス（CSV or JSON）がフィクスチャとして保存され、品目・卸売価格・調査年月の
  列位置が確定していること。
- 自動化度: probe実行はGitHub Actions（`workflow_dispatch`）で自動化済み。**候補URLの選定・
  統計表IDの特定は人力が必要**（e-Statの統計表検索UIを人が操作する必要がある）。

### A-2. 東京都中央卸売市場「市場統計情報」のデータURL特定
- 内容: 東京都中央卸売市場のサイトはJS動的CMSであり、HTMLを直接パースする方式は使えない。
  ブラウザの開発者ツール等でXHR/fetch通信を確認し、実際にデータを返すAPI/documentsエンドポイント
  （CSV/Excelの直リンクを含む）を特定する必要がある。
- 手段: これは**人力での事前調査が必須**（動的CMSの通信をブラウザで観察する作業は、現状の
  probeワークフロー＝単純HTTP取得では代替できない）。エンドポイントが判明したら、probeの候補URLに
  追加してGitHub Actionsから到達性・レスポンス形式を再確認する。
- 完了条件: 日次・週次データを返す実URLが判明し、フィクスチャ化されていること。
- 自動化度: 低い（初回のエンドポイント調査は人力。判明後の定期取得は自動化可能）。

### A-3. パーサ実装・並行運用・切り替え
- 内容: A-1/A-2で確定したレイアウトに対応する新アダプタ（`estatAdapter` の実装完成、
  および必要なら `tokyoShijoAdapter` を新設）を、既存の `vegetanAdapter` と同じ
  fetch→normalize インターフェースで実装する。
- 完了条件: 新アダプタが `npm test` のフィクスチャテストを通過し、`config/items.json` の
  対象品目の `source` を切り替えても既存ページのレイアウト・機能（平年比・ランキング等）が
  壊れないことをE2Eテスト（`test/build.test.mjs`）で確認できること。
- 移行判断: 政府統計への完全移行後は `docs/legal-notes.md` のリスク評価を「解消」に更新する。
  ベジ探の平年比データ等、政府統計に相当データがない指標は代替算出方法を検討するか、当該指標のみ
  ベジ探を参照し続けるかを判断する（機構への利用連絡の回答を踏まえて決定）。
- 自動化度: 実装・テストはこれまで通りコーディングエージェントで自動化可能。**移行の可否判断・
  機構への連絡結果待ちは人力の意思決定が必要**。

## フェーズB: 収益化

**目的**: 月間収益 10万円までのKPI到達。**現実的な期間として、SEOでの到達には6〜12ヶ月程度を
見込む（保証ではない）。**

### B-1. Google AdSense
- 作業: (1) AdSenseアカウント作成・サイト審査申請、(2) 承認後 `config/site.json` の
  `adsenseClientId` / `adsenseSlotTop` / `adsenseSlotItem` を設定。
- 完了条件: 設定後の次回ビルドから `src/templates/layout.mjs` の `adSlot()` / `adsenseHead()` が
  非空文字列を検知して広告タグを自動出力する（**コード変更不要、設定のみで完結する設計**）。
- 自動化度: 審査申請・承認待ちは**完全に人力**（Googleの審査プロセス）。設定後の反映はビルド
  スクリプトが自動で行う。

### B-2. アフィリエイト（Amazon/楽天等、買い時野菜×レシピ・保存容器等の関連商品）
- 作業: A8.net / Amazonアソシエイト / 楽天アフィリエイト等への登録、審査通過後に
  `config/site.json` の `affiliate.enabled` / `affiliate.links` を設定。
- **現状の実装ギャップ（重要）**: `config/site.json` の `affiliate` フィールドは現時点では
  「設定を保持するだけの箱」であり、`src/templates/layout.mjs` にアフィリエイトリンクを実際に
  レンダリングするロジックは未実装（`docs/architecture.md` に「拡張可能」と記載の通り、フックのみ）。
  AdSenseと異なり、**設定するだけでは広告は出ない**。品目ページの `buyKeyword`
  （`config/items.json`）と紐づけて「買い時野菜のレシピ」「保存容器」等の関連商品リンクを表示する
  コンポーネントの実装が必要。
- 完了条件: 品目ページ・買い時ランキングページにアフィリエイトリンクが表示され、
  `affiliate.enabled=false` 時は一切出力されないこと（AdSenseと同様のオプトイン設計）。
- 自動化度: 登録・審査は人力。リンクの表示ロジック実装はコーディングで自動化可能。
  商品選定（どのレシピ・容器を紐づけるか）は人力の判断が必要（機械的な自動生成は品質リスクあり）。

### B-3. KPI試算（10万円/月まで）
- 前提: RPM（広告表示1000回あたり収益）はAdSense単体で概ね300〜500円程度と仮定
  （サイト内容・季節・広告主の入札状況により変動するため、あくまで概算）。
- 試算: 月10万円 ÷ RPM 300〜500円 × 1000 = **月間20万〜33万PV程度**が必要
  （アフィリエイト収益を含めればPV要件は下がりうるが、コンバージョン率に依存するため保守的に
  広告収益のみで試算）。
- 到達期間: 新規ドメイン・無名サイトのSEOでの成長曲線を踏まえ、**現実的には6〜12ヶ月程度**を
  見込む（保証ではなく、内部リンク構造・更新頻度・被リンク獲得次第で前後する）。
- 自動化度: PVの成長はSEO・コンテンツ拡充（フェーズC）に依存し人力の関与が大きい。KPIダッシュボード
  化（Google Search Console連携等）は今後の課題。

## フェーズC: 拡張

### C-1. 品目追加（果物・水産）
- 内容: ベジ探または移行後の一次統計に果物・水産物のデータがあれば、`config/items.json` に
  品目を追加。既存の野菜と同じ正規化パイプラインを再利用できる見込み。
- 完了条件: 新品目のページが既存品目と同じ機能（日次/月次チャート・平年比・ランキング）を持つこと。
- 自動化度: 高い（既存パイプラインの再利用が中心）。品目ごとのメタデータ（旬・産地等の説明文）は
  人力での執筆が必要。

### C-2. 都市別小売価格ページ
- 内容: `kouri_cyousa` データ（東京都中央卸売市場「小売価格調査」相当。フィクスチャ実装済み:
  `test/fixtures/vegetan/032〜048`）を用いた都市別・品目別の小売価格ページを新設。
- 完了条件: フィクスチャに対するパーサ（`src/lib/xlsx.mjs` は既にラウンドトリップ確認済み）を
  `src/lib/vegetan.mjs` に追加し、新ページテンプレートをビルドスクリプトに追加。
- 自動化度: 高い（既存フィクスチャがあるため実装のみで着手可能）。

### C-3. B2B向け価格データAPI・メール配信（有料）
- 内容: 蓄積した `data/items/*.json` をベースに、JSON API（静的ファイル配信で可）または
  日次メールダイジェストを有料提供する。
- 完了条件: API利用規約・料金プランの策定、認証（APIキー）方式の実装。
- 自動化度: データ生成自体は自動化済み。**料金プラン策定・利用規約整備・決済導線は人力が必須**。
  ベジ探由来データを含む場合は `docs/legal-notes.md` のリスク評価（データベース著作物性）を
  再検討する必要がある（有償再配布は無償閲覧より機構との関係整理の必要性が高まる可能性）。

### C-4. 多言語化
- 内容: 英語版ページの追加（訪日外国人・海外の日本食材バイヤー向け）。
- 完了条件: `/en/` 配下に主要ページの翻訳版を生成。
- 自動化度: 翻訳はLLM支援で効率化できるが、**専門用語（品目名・単位）の正確性チェックは人力が必要**。
